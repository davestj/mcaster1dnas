## Project Overview

**Mcaster1DNAS** (Digital Network Audio Server) is a fork/evolution of Icecast 2.x, extended with:
- ICY 2.2 extended metadata protocol
- YAML configuration (libyaml, in addition to legacy XML)
- Per-listener `ssl: true/false` enforcement
- Static mount point definitions with ICY 2.2 field support
- Song history / now-playing JSON REST API (`/songdata`)
- **macOS-native Qt6 GUI** (`Mcaster1DNAS.app`) ‚Äî Config Manager, SSL cert tab, ICY2.2 structured fields, Web UI/Admin launch
- Windows-native GUI (MFC, VS2022) + Windows Service
- NSIS installer for Windows distribution

**Critical macOS-specific gotchas:**

1. **`config_clear()` aborts when server not yet started** ‚Äî `config_clear()` calls `global_lock()` which calls `thread_mutex_lock_c()` ‚Üí `abort()` because the server's pthread mutex is uninitialized until `server_init()` runs. Never call `config_clear()` on a local `mc_config_t` in the GUI. `ConfigDialog::loadFromFile()` uses `config_parse_file()` directly and deliberately omits `config_clear()`.

2. **Config discovery from `.app` bundle** ‚Äî `open` sets CWD to `/`; the binary lives at `.../Mcaster1DNAS.app/Contents/MacOS/`. `discoverConfig()` walks 6 directory levels up from the binary path to find `mcaster1dnas.yaml` at the repo root.

3. **Codesign after every rebuild** ‚Äî Apple Silicon caches the code-directory; after `cmake --build` replaces the binary, run `codesign --force --sign - macos/build-qt/Mcaster1DNAS.app` or the app gets SIGKILL on launch.

4. **`discoverServerUrl()` / Web UI buttons** ‚Äî reads live config via `config_get_config_unlocked()` when server running, or parses config file directly (without `config_clear()`). Maps `0.0.0.0` / `::` / `*` to `127.0.0.1`. Prefers HTTPS port over HTTP.

5. **`#ifdef HAVE_XSLTSAVERESULTTOSTRING` in xslt.c** ‚Äî on libxml2 ‚â• 2.12, the old `xmlOutputBufferCreateIO` + `xsltSaveResultTo` path crashes in `xmlBufAdd ‚Üí _platform_memmove`. Fixed by using `xsltSaveResultToString` when available (always true with Homebrew libxslt ‚â• 1.1.18).

6. **`config_parse_file()` needs logging rwlock** ‚Äî `config_parse_file()` calls `log_write()` internally, which acquires `_logger_rwl`. That rwlock is NULL until `init_log_subsys()` runs (normally called inside `initialize_subsystems()` when the server starts). Calling `config_parse_file()` from the GUI thread (e.g. `ConfigDialog::loadFromFile()`, `discoverServerUrl()`) before the server starts ‚Üí `KERN_INVALID_ADDRESS at 0x0` crash.
   **Pattern:** `init_log_subsys()` ‚Üí `config_parse_file(...)` ‚Üí `log_shutdown()`. Include `"logging.h"` and `"log/log.h"` in the `extern "C"` block. `log_shutdown()` itself had a latent bug fixed in `src/log/log.c` (see macOS Source Fixes below).

7. **`OverviewHeader` banner scaling** ‚Äî `header-banner-v2.png` (1926√ó142 px) is pre-scaled to half height at load time: `m_px.scaledToHeight(m_px.height()/2, Qt::SmoothTransformation)` ‚Üí 71 px. Drawn to fill widget width via `p.drawPixmap(0, 0, W, H, m_px)`. Badge dimensions: bW=50, bH=61, bY=(H-bH)/2 ‚Äî proportional to 71px banner. These numbers must stay paired ‚Äî scaling one without the other makes the badge look over- or under-sized.

### SSL Certificate: `ssl/localhost.pem`
- CN=localhost, SANs: `DNS:localhost`, `IP:127.0.0.1`
- Format: PRIVATE KEY block first, then CERTIFICATE block (combined PEM)
- 825-day validity, RSA 2048
- Generated with Git Bash + Git's bundled openssl:
  ```bash
  MSYS_NO_PATHCONV=1 openssl req -x509 -newkey rsa:2048 -nodes -days 825 \
    -subj "/C=US/ST=TX/L=Dallas/O=Mcaster1DNAS/OU=Local/CN=localhost" \
    -addext "subjectAltName=DNS:localhost,IP:127.0.0.1" \
    -keyout key.pem -out cert.pem
  cat key.pem cert.pem > ssl/localhost.pem
  ```
  > `MSYS_NO_PATHCONV=1` is **required** to prevent Git Bash from converting `-subj "/C=..."` to a Windows path.

## Windows Service Architecture

**Binary:** `mcaster1Service.exe` (NOT `mcaster1.exe`)

**Service registration (NSIS):**
```
sc create Mcaster1DNAS binPath= "\"...\mcaster1Service.exe\" \"...\Mcaster1DNAS\"" ...
```

**Starting/stopping (requires admin):**

```powershell
Start-Process powershell -Verb RunAs -ArgumentList "-Command sc.exe start Mcaster1DNAS" -Wait
Start-Process powershell -Verb RunAs -ArgumentList "-Command sc.exe stop Mcaster1DNAS" -Wait
```

## Critical Bugs Fixed (Historical)

### 1. `src/params.c` ‚Äî MSVC struct initializer bug
MSVC's struct positional initializers mapped fields incorrectly in `mc_http_printf` and
`mc_params_printf` (name‚Üíflags, content‚Üíname_len), leaving `name=NULL` and silently dropping
ALL HTTP response headers on Windows ‚Äî including `WWW-Authenticate`, breaking admin auth 401.

**Fix:** `memset(&hdr, 0, sizeof hdr)` + explicit field assignments in all MSVC blocks.

### 2. `windows/mcaster1Service.cpp` ‚Äî wrong default config
`ServiceMain` defaulted to `mcaster1.xml`; config is `mcaster1dnas-console.yaml`.

**Fix:** `argv2[2] = (char*)"mcaster1dnas-console.yaml"`

### 3. `windows/Mcaster1Win.cpp` ‚Äî no working directory set
Relative paths in YAML config (`./logs`, `./web`, `./ssl`) resolved to wrong location when
launched from shortcuts or by the SCM.

**Fix:** `SetCurrentDirectoryA(exedir)` added as first thing in `InitInstance()`.

### 4. NSIS ‚Äî wrong service binary
Installer used `mcaster1.exe` (console app) as service binary. Console app never calls
`StartServiceCtrlDispatcher`; SCM times out after 30s.

**Fix:** `sc create ... binPath= "\"$INSTDIR\mcaster1Service.exe\" \"$INSTDIR\""`.

### 5. `build-installer.ps1` ‚Äî em dash encoding crash
Em dash `‚Äî` characters (UTF-8 E2 80 94, byte 0x94 = Windows-1252 `"`) caused
PowerShell 5.1 to treat them as string terminators ‚Üí cascading parse errors.

**Fix:** All em dashes replaced with plain ASCII hyphens. File must be saved UTF-8 without BOM.

### 6. PowerShell drive reference ambiguity
`"text $VarName:"` is parsed as a drive letter reference.

**Fix:** Use `"text ${VarName}:"` with braces.

## Windows Gotchas

- **MSVC:** No GNU designated initializers (`{.field = value}`). Use `memset + explicit assignment`.
- **File locking:** Use `_fsopen(path, "rb", _SH_DENYNO)` to read files held open by the server. `fopen`/`fopen_s` will fail.
- **libyaml on Windows:** Use `yaml_parser_set_input_string`, NOT `set_input_file` (CRT DLL boundary issue with FILE*).
- **ResizableLib:** `IDC_STATIC` (0xFFFF) is not anchorable ‚Äî use unique resource IDs.
- **OpenSSL in Git Bash:** Always prefix with `MSYS_NO_PATHCONV=1` when passing `-subj "/C=..."` to prevent path conversion.
- **Program Files writes:** Require UAC elevation. Use `Start-Process ... -Verb RunAs -Wait`.
- **`sc start` requires admin.** Use `Start-Process ... -Verb RunAs`.
