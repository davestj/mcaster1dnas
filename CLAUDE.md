## Project Overview

**Mcaster1DNAS** (Digital Network Audio Server) is a fork/evolution of Icecast 2.x, extended with:
- ICY 2.2 extended metadata protocol
- YAML configuration (libyaml, in addition to legacy XML)
- Per-listener `ssl: true/false` enforcement
- Static mount point definitions with ICY 2.2 field support
- Song history / now-playing JSON REST API (`/songdata`)
- Windows-native GUI (MFC, VS2022) + Windows Service
- NSIS installer for Windows distribution

## Build System

### Windows (VS2022 Professional)

**Full build + installer:**
```powershell
cd windows\installer
powershell -ExecutionPolicy Bypass -File build-installer.ps1 -Config Debug
```

**Flags:**
- `-SkipBuild`   — skip MSBuild, go straight to NSIS (DLLs must already be present)
- `-SkipImages`  — skip PNG→BMP conversion (use existing BMPs)
- `-SkipNSIS`    — dry-run, no installer output

**Build individual project:**
```powershell
& "C:\Program Files\Microsoft Visual Studio\2022\Professional\MSBuild\Current\Bin\MSBuild.exe" `
    "C:\Users\dstjohn\dev\00_mcaster1.com\mcaster1dnas\windows\Mcaster1DNAS.sln" `
    /p:Configuration=Debug /p:Platform=x64 /t:Build /v:minimal
```

**Output directory:** `windows\x64\Debug\`

**vcpkg root:** `C:\Users\dstjohn\dev\vcpkg`
**vcpkg DLLs:** `%USERPROFILE%\dev\vcpkg\installed\x64-windows\bin\`
**vcpkg openssl:** `%USERPROFILE%\dev\vcpkg\installed\x64-windows\tools\openssl\openssl.exe`

> **Known gotcha — missing DLLs after build:**
> MSBuild post-build events (`copy-vcpkg-deps.ps1`) report "22 DLLs copied" but some may not

> run: `cp $vcpkg\installed\x64-windows\bin\*.dll windows\x64\Debug\`

## Configuration Files

### `mcaster1dnas-console.yaml` (repo root)
Used by `mcaster1.exe` (console) and `mcaster1Service.exe` (Windows Service).
- Ports 9330 (plain) + 9443 (SSL)
- `ssl-certificate: ./ssl/localhost.pem`
- Log level: `info`
- All paths relative to working directory (set by `_chdir(argv[1])` in service, or CWD for console)
- **Change `CHANGE_ME_*` passwords before distributing or running the service**

### `windows/mcaster1dnas.yaml` (source) / installed as `mcaster1dnas.yaml`
Used by `mcaster1win.exe` (GUI application).
- Ports 9033 (plain) + 9344 (SSL) — sidecar ports, avoids collision with service
- `ssl-certificate: ./ssl/localhost.pem`
- All paths relative: `basedir: "."`, `logdir: "./logs"`, `webroot: "./web"`, etc.
- Working directory anchored by `SetCurrentDirectoryA(exedir)` at app startup

### SSL Certificate: `ssl/localhost.pem`
- CN=localhost, SANs: `DNS:localhost`, `IP:127.0.0.1`
- Format: PRIVATE KEY block first, then CERTIFICATE block (combined PEM)
- 825-day validity, RSA 2048
- Generated with Git Bash + Git's bundled openssl:
  ```bash
  MSYS_NO_PATHCONV=1 openssl req -x509 -newkey rsa:2048 -nodes -days 825 \
    -subj "/C=US/ST=TX/L=Dallas/O=Mcaster1DNAS/OU=Local/CN=localhost" \
    -addext "subjectAltName=DNS:localhost,IP:127.0.0.1" \
    -keyout key.pem -out cert.pem
  cat key.pem cert.pem > ssl/localhost.pem
  ```
  > `MSYS_NO_PATHCONV=1` is **required** to prevent Git Bash from converting `-subj "/C=..."` to a Windows path.

## Windows Service Architecture

**Binary:** `mcaster1Service.exe` (NOT `mcaster1.exe`)

**Service registration (NSIS):**
```
sc create Mcaster1DNAS binPath= "\"...\mcaster1Service.exe\" \"...\Mcaster1DNAS\"" ...
```
The second argument is the **install directory** (not the config file). `mcaster1Service.cpp`
calls `_chdir(argv[1])` then defaults the config to `mcaster1dnas-console.yaml`.

**`mcaster1Service.cpp` key facts:**
- `main()`: `_chdir(argv[1])` → `StartServiceCtrlDispatcher`
- `ServiceMain()`: argv[1] from SCM args overrides default config; default = `mcaster1dnas-console.yaml`
- Must call `SetServiceStatus(SERVICE_RUNNING)` within ~30s or SCM times out

**Starting/stopping (requires admin):**
```powershell
Start-Process powershell -Verb RunAs -ArgumentList "-Command sc.exe start Mcaster1DNAS" -Wait
Start-Process powershell -Verb RunAs -ArgumentList "-Command sc.exe stop Mcaster1DNAS" -Wait
```

## WinUI (mcaster1win.exe) Architecture

**`Mcaster1Win.cpp` — `InitInstance()`:**
1. `SetCurrentDirectoryA(exedir)` — anchors CWD to exe dir (CRITICAL — must be first)
2. `ParseMcasterCmdLine()` — parses `-c`, `--ssl-*`, etc.
3. Config file discovery: `-c` arg → `<exedir>/mcaster1dnas.yaml` → `./mcaster1dnas.yaml`
4. `AfxOleInit()`, `AfxEnableControlContainer()`
5. `m_pMainWnd = &dlg; dlg.DoModal()`

**Tabs (dialog):**
- **Stats**: live listener/source stats, ICY metadata, uptime clock, system clock
- **Config**: YAML/XML config editor with `notepad.exe` integration
- **Log**: Live log viewer (`LogTab.cpp` — polls via `_fsopen` with `_SH_DENYNO`)
- **SSL**: Certificate generator, SslGen.cpp wrapper

**`LogTab.cpp` — live log tailing:**
- `_fsopen(path, "rb", _SH_DENYNO)` — shares read access with server's open log file
- Timer fires every 2000ms; appends new content; color-codes error log lines by level
- Returns silently if `cfg->log_dir == NULL` (server not started)

## Critical Bugs Fixed (Historical)

### 1. `src/params.c` — MSVC struct initializer bug
MSVC's struct positional initializers mapped fields incorrectly in `mc_http_printf` and
`mc_params_printf` (name→flags, content→name_len), leaving `name=NULL` and silently dropping
ALL HTTP response headers on Windows — including `WWW-Authenticate`, breaking admin auth 401.

**Fix:** `memset(&hdr, 0, sizeof hdr)` + explicit field assignments in all MSVC blocks.

### 2. `windows/mcaster1Service.cpp` — wrong default config
`ServiceMain` defaulted to `mcaster1.xml`; config is `mcaster1dnas-console.yaml`.

**Fix:** `argv2[2] = (char*)"mcaster1dnas-console.yaml"`

### 3. `windows/Mcaster1Win.cpp` — no working directory set
Relative paths in YAML config (`./logs`, `./web`, `./ssl`) resolved to wrong location when
launched from shortcuts or by the SCM.

**Fix:** `SetCurrentDirectoryA(exedir)` added as first thing in `InitInstance()`.

### 4. NSIS — wrong service binary
Installer used `mcaster1.exe` (console app) as service binary. Console app never calls
`StartServiceCtrlDispatcher`; SCM times out after 30s.

**Fix:** `sc create ... binPath= "\"$INSTDIR\mcaster1Service.exe\" \"$INSTDIR\""`.

### 5. `build-installer.ps1` — em dash encoding crash
Em dash `—` characters (UTF-8 E2 80 94, byte 0x94 = Windows-1252 `"`) caused
PowerShell 5.1 to treat them as string terminators → cascading parse errors.

**Fix:** All em dashes replaced with plain ASCII hyphens. File must be saved UTF-8 without BOM.

### 6. PowerShell drive reference ambiguity
`"text $VarName:"` is parsed as a drive letter reference.

**Fix:** Use `"text ${VarName}:"` with braces.

## Windows Gotchas

- **MSVC:** No GNU designated initializers (`{.field = value}`). Use `memset + explicit assignment`.
- **File locking:** Use `_fsopen(path, "rb", _SH_DENYNO)` to read files held open by the server. `fopen`/`fopen_s` will fail.
- **libyaml on Windows:** Use `yaml_parser_set_input_string`, NOT `set_input_file` (CRT DLL boundary issue with FILE*).
- **ResizableLib:** `IDC_STATIC` (0xFFFF) is not anchorable — use unique resource IDs.
- **OpenSSL in Git Bash:** Always prefix with `MSYS_NO_PATHCONV=1` when passing `-subj "/C=..."` to prevent path conversion.
- **Program Files writes:** Require UAC elevation. Use `Start-Process ... -Verb RunAs -Wait`.
- **`sc start` requires admin.** Use `Start-Process ... -Verb RunAs`.
