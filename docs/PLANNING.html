VersionPlatformBugFix
2.5.3-beta macOS src/xslt.c — crash in xmlBufAdd → _platform_memmove on every XSLT request (libxml2 ≥ 2.12) Replaced xmlOutputBufferCreateIO + xsltSaveResultTo with xsltSaveResultToString guarded by HAVE_XSLTSAVERESULTTOSTRING
2.5.2-dev macOS src/cfgfile.h:422 — xmlErrorPtr vs const xmlError * (libxml2 2.12+ API change) Changed parameter type in config_xml_parse_failure()
2.5.2-dev macOS src/slave.c:802 — size_t return from CURLOPT_XFERINFOFUNCTION callback (expects int) Changed icecurl_server_running() return type to int
2.5.2-dev macOS src/xslt.c — double-free after xsltSaveResultTo on libxml2 ≥ 2.12 (htmlSaveFileTo now closes buffer internally) Added #if LIBXML_VERSION < 21200 guard around xmlOutputBufferClose(buf)
2.5.2-dev macOS Qt6 GUI: ConfigDialog::loadFromFile() calls config_clear() → global_lock() → abort() when server not yet started Removed config_clear() call; documented why (mutex uninitialized until server_init() runs)
2.5.2-dev Windows src/params.c — MSVC struct positional initializer bug dropped all HTTP response headers (incl. WWW-Authenticate) memset(&hdr, 0, sizeof hdr) + explicit field assignments
2.5.2-dev Windows mcaster1Service.cpp — wrong default config (mcaster1.xml instead of mcaster1dnas-console.yaml) Fixed default config string in ServiceMain()
2.5.2-dev Windows NSIS installer — used mcaster1.exe as service binary (never calls SCM dispatcher) Changed to mcaster1Service.exe
2.5.3-beta macOS Qt6 GUI Config button crash #1 — ConfigDialog::loadFromFile() and discoverServerUrl() call config_parse_file() which calls log_write(); the logging rwlock _logger_rwl is NULL before the server starts → KERN_INVALID_ADDRESS at 0x0 Wrap config_parse_file() with init_log_subsys() + log_shutdown(). Add logging.h + log/log.h to each file's extern "C" block.
2.5.3-beta macOS / All Config button crash #2 — src/log/log.c log_close() latent bug — without log_callback, log_close() iterated all 20 pre-allocated loglist entries and called _log_close_internal() on unused ones (mutex=NULL). _log_close_internal() contract: “enter with q lock” — but log_close() never called _lock_q() on the non-callback path → KERN_INVALID_ADDRESS at 0x10 (offset of sys_mutex in mutex_t) In log_close(): (1) if (loglist[log_id].in_use == 0) break; — skip unused entries; (2) _lock_q(log_id); before _log_close_internal() to satisfy the function contract.
2.5.3-beta macOS Qt6 GUI Overview tab banner over-scaling — header-banner-v2.png (1926×142 px) rendered at full natural height dominated the Overview tab; status indicator badge sized independently caused mismatched proportions when either was changed without the other Pre-scale banner at load: m_px.scaledToHeight(m_px.height()/2) → 71 px. Draw scaled to widget width: p.drawPixmap(0,0,W,H,m_px). Badge proportionally matched: bW=50, bH=61, bY=(H-bH)/2.
